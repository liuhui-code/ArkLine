# ArkLine v0.1.0

First public engineering snapshot of ArkLine.

## Summary

ArkLine is a lightweight, Windows-first ArkTS IDE shaped around code reading,
navigation, `lint` / `format`, and Git review in an IDEA-inspired shell.

This release is an MVP snapshot rather than a finished product release. It is
meant to establish the desktop shell, editing flow, packaging direction, and
the first ArkTS query loop.

## Highlights

- IDEA-inspired desktop shell with Project, Search, editor tabs, status bar, and bottom tool windows
- CodeMirror-based editor for `.ets`, `.ts`, and `.json5`
- Recent Projects, Recent Files, Search Everywhere, Quick Open, and Find Action
- Git-owned bottom diff review workflow
- Terminal MVP for configured validation commands
- First semantic-query loop wiring for:
  - `Ctrl+B` Go to Definition
  - `Ctrl+Click` definition navigation
  - `Ctrl+Space` completion
  - `Alt+F7` Find Usages
- Windows packaging baseline through Tauri v2

## Platform notes

### Windows

Windows is the primary target for ArkLine.

Expected usage path:

1. install the packaged build
2. ensure WebView2 Runtime is available
3. open an ArkTS workspace with `File -> Open Project...`

### macOS

macOS support is currently intended for development and verification, not as the
main packaged end-user target.

## Known limitations

- ArkTS language-service integration is not complete yet
- definition, completion, and usages are still MVP-level
- lint / format are still shell-driven integration points rather than deep IDE services
- Git support is still much lighter than IntelliJ IDEA Git tooling
- terminal support is intentionally narrow and is not a full PTY terminal
- the packaged Windows installer still needs final real-machine validation

## Recommended audience for this release

- developers following ArkLine’s architecture direction
- contributors validating the IDEA-like shell and workflow
- ArkTS tooling experiments where a lightweight Windows-first shell is useful

## Not yet promised by this release

- production-grade ArkTS semantic accuracy
- full IntelliJ-class refactoring or indexing
- complete Git client behavior
- cross-platform parity as a product goal
